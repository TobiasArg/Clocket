## ADDED Requirements

### Requirement: Market quote endpoint accepts only canonical requests
The backend SHALL expose `GET /api/market/quote` with `assetType` limited to `stock` or `crypto` and `ticker` normalized to an uppercase provider-safe symbol.

#### Scenario: Valid stock quote request
- **WHEN** a client requests `/api/market/quote?assetType=stock&ticker=aapl`
- **THEN** the backend MUST normalize the ticker to `AAPL` and request a stock quote

#### Scenario: Valid crypto quote request
- **WHEN** a client requests `/api/market/quote?assetType=crypto&ticker=btc`
- **THEN** the backend MUST normalize the ticker to `BTC` and request a crypto USD exchange rate

#### Scenario: Invalid asset type
- **WHEN** `assetType` is not `stock` or `crypto`
- **THEN** the backend MUST return HTTP 400 with a canonical error payload

#### Scenario: Invalid ticker
- **WHEN** `ticker` does not match the allowed symbol pattern
- **THEN** the backend MUST return HTTP 400 with a canonical error payload

#### Scenario: Unsupported method
- **WHEN** the endpoint receives a non-GET method
- **THEN** the backend MUST return HTTP 405 and set `Allow: GET`

### Requirement: Market quote success response is canonical
The backend SHALL return a normalized quote payload that the frontend can consume without provider-specific parsing.

#### Scenario: Successful stock response
- **WHEN** Alpha Vantage returns a valid `GLOBAL_QUOTE` payload
- **THEN** the backend MUST return `assetType`, `ticker`, `currentPrice`, `source`, `asOf`, `dailyPctFromProvider`, `lastRefreshed`, `timezone`, `bid`, and `ask`

#### Scenario: Successful crypto response
- **WHEN** Alpha Vantage returns a valid `CURRENCY_EXCHANGE_RATE` payload
- **THEN** the backend MUST return `assetType`, `ticker`, `currentPrice`, `source`, `asOf`, `dailyPctFromProvider`, `lastRefreshed`, `timezone`, `bid`, and `ask`

#### Scenario: Response timestamp
- **WHEN** the backend returns a successful quote
- **THEN** `asOf` MUST be an ISO timestamp generated by the backend at response time

### Requirement: Market quote error response is canonical
The backend SHALL return canonical error payloads with enough metadata for the frontend to decide whether to keep a stale local snapshot.

#### Scenario: Provider throttling
- **WHEN** Alpha Vantage returns an `Information` or `Note` throttling payload
- **THEN** the backend MUST return HTTP 429 with `code: "THROTTLED"`, `status: 429`, `retryable: false`, and `stalePolicy: "KEEP_LAST_SNAPSHOT"`

#### Scenario: Invalid provider symbol
- **WHEN** Alpha Vantage rejects or cannot resolve the ticker
- **THEN** the backend MUST return HTTP 422 with `code: "INVALID_SYMBOL"`, `status: 422`, `retryable: false`, and `stalePolicy: "KEEP_LAST_SNAPSHOT"`

#### Scenario: Provider parse failure
- **WHEN** Alpha Vantage returns a payload missing required quote fields
- **THEN** the backend MUST return HTTP 502 with `code: "PARSE_ERROR"`, `status: 502`, and `stalePolicy: "KEEP_LAST_SNAPSHOT"`

#### Scenario: Network or transient provider failure
- **WHEN** Alpha Vantage cannot be reached or returns a transient HTTP status
- **THEN** the backend MUST return a canonical error payload containing `error`, `code`, `status`, `retryable`, and `stalePolicy`

### Requirement: Alpha Vantage provider adapter preserves current safeguards
The backend SHALL keep provider retry, transient failure classification, throttling detection, and minimum request interval behavior while making configuration testable.

#### Scenario: Retryable provider status
- **WHEN** Alpha Vantage returns a retryable HTTP status and retry budget remains
- **THEN** the provider adapter MUST retry according to the configured retry policy before returning an error

#### Scenario: Non-retryable provider payload error
- **WHEN** Alpha Vantage returns a semantic provider error such as `Error Message`
- **THEN** the provider adapter MUST not retry and MUST map the payload to a canonical provider error
